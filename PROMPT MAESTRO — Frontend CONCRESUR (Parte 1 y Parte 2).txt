CONVENCIONES GLOBALES OBLIGATORIAS

--- Nomenclatura de componentes ---
Base*     → átomos reutilizables sin lógica de negocio
             BaseButton, BaseInput, BaseSelect, BaseModal,
             BaseTable, BaseBadge, BaseCard
The*      → únicos de layout: TheSidebar, TheNavbar
[Módulo]* → específicos del dominio: VentaForm, KanbanCard

--- Composition API ---
Siempre <script setup> en JS plano (sin TypeScript).
Extraer lógica repetida a composables, las vistas son delgadas.
Un composable por responsabilidad.

--- Axios ---
Una instancia en src/api/axios.js:
  baseURL = import.meta.env.VITE_API_BASE_URL
  Interceptor request: adjuntar Authorization: Bearer {token}
  Interceptor response:
    401 → limpiar authStore + router.push('/login')
    403 → toast.error('Sin permisos para esta acción')
    422 → retornar errores al llamador (para VeeValidate)
    500 → toast.error('Error interno del servidor')
Un archivo por módulo en src/api/ que exporta funciones puras.
Nunca llamar axios directamente desde una vista.

--- Pinia stores ---
Un store por módulo con estructura: state / getters / actions.
Las actions hacen la llamada API y mutan el state.
Usar try/catch en todas las actions con toast en el catch.
authStore persiste { token, user } en localStorage.

--- Permisos ---
Directiva v-permission="'ventas.ventas.crear'" que oculta
el elemento del DOM si el usuario no tiene ese permiso.
Guard de navegación verifica meta.permiso antes de entrar.
Los permisos vienen del payload de GET /auth/me al hacer login.
Claves de permiso exactas del backend:
  sistema.usuarios.ver, sistema.usuarios.crear
  sistema.roles.ver, sistema.roles.editar
  catalogos.productos.ver, catalogos.productos.crear
  inventario.stock.ver, inventario.ajuste.crear
  ventas.cotizaciones.ver, ventas.cotizaciones.crear
  ventas.ventas.ver, ventas.ventas.crear
  ventas.ventas.confirmar

--- Formularios ---
VeeValidate + Yup en todos los formularios.
Mostrar errores inline debajo de cada campo.
Deshabilitar botón submit mientras hay request en vuelo.
Nunca llamar a la API sin haber validado antes.

--- Feedback visual ---
Toast de éxito/error en toda operación CRUD.
Spinner en tablas y formularios durante carga.
Estado vacío con mensaje cuando no hay datos.
Modal de confirmación antes de desactivar registros.

--- Responsividad ---
Sidebar colapsable en móvil con botón hamburguesa.
Tablas con scroll horizontal en pantallas pequeñas.
Formularios: 1 columna en móvil, 2 columnas en desktop.

--- Stores ---
authStore.js
  state: { token: null, user: null, permisos: [] }
  getters:
    isAuthenticated: !!token
    hasPermiso(clave): permisos.includes(clave)
  actions:
    login(credentials)
      → POST /auth/login
      → guardar token en localStorage + state
      → guardar user y permisos desde response
    logout()
      → POST /auth/logout
      → limpiar state + localStorage
      → router.push('/login')
    initAuth()
      → al montar App.vue: restaurar token desde localStorage
      → GET /auth/me para obtener user + permisos actuales
      → si falla 401: limpiar y redirigir a /login

sistemaStore.js
  state: { usuarios: [], roles: [], permisos: [], personal: [] }
  actions:
    cargarUsuarios(), crearUsuario(data),
    actualizarUsuario(id, data), desactivarUsuario(id),
    cargarRoles(), crearRol(data),
    actualizarRol(id, data), desactivarRol(id),
    sincronizarPermisos(rol_id, permiso_ids),
    cargarPermisos(),
    cargarPersonal(filtros), crearPersonal(data),
    actualizarPersonal(id, data), desactivarPersonal(id)

catalogosStore.js
  state: { productos: [], variantesMap: {} }
  actions:
    cargarProductos(), crearProducto(data),
    actualizarProducto(id, data), desactivarProducto(id),
    cargarVariantes(producto_id), crearVariante(data),
    actualizarVariante(id, data), desactivarVariante(id)
